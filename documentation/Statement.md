Requirements Understanding and Interpretation
 
Functional Requirements
Do what’s only specified, not more, not less.
 
For Front-end visual layout, use Bootstrap. Remember the site should be Responsive for Mobile and Desktop. 
For Front-end interaction, use only jQuery for interaction and jQuery Ajax for data consumption
For Front-end data-bindings, use only Handlerbars
For Back-end use Code Igniter PHP Framework, use Routes for friendly URLs and Active Record for data consumption.
The Application will have two tiers
Client: This will be only Html, Css, Javascript.
API: This will be a RESTful JSON API that will be used by the Client. (For this workshop we won’t work with Security and Authentication of the API)
Version every step of the process. Make sure to follow an incremental flow instead of multitasking.
Every commit must make sense, and should be attacking a single problem.
Add a README.md file and put there what you consider relevant to get the app up and running in no time. 
Add a CHANGELOG.md and put there the progress you are making day by day.

All Articles Screen
 
List All articles from the API service
Each Article should accept a click to go to its details
In the top of the page, offer an option for creating a new Article. This should take the user to the details page.
Each Article will have an image, a title, and a description.
 
Article Detail
 
Choose an image from your device, either in desktop finder or a mobile photo gallery.
Let the user fill out a tittle and a description.
Let the user save a new Article, edit or remove a previous selected Article from the All Articles Screen.
Use te API service to handle creation, update, and removal of Articles.

Para utilizar Back-end Code Igniter PHP Framework, utilice Rutas para URLs amigables y Active Record para el consumo de datos.
La aplicación tendrá dos niveles
Cliente: Esto será sólo Html, Css, Javascript.
API: Esta será una API RESTful JSON que será utilizada por el Cliente. (Para este taller no trabajaremos con seguridad y autenticación de la API)
Versión cada paso del proceso. Asegúrese de seguir un flujo incremental en lugar de multitarea.
